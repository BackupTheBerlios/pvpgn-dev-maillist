Comment:
Adds some additional info to the Servers XML status file like:

User:
+<country>
+<clientlang> if exists
+<realm> if exists
+<charname> if exists
+<game> if exists

Games:
+<numplayer>
+<realm> if exists

These additional fields should be unproblematic to existing XML solutions/processing.

Haven't changed the .dat format for now.
I don't require it, but for consistency reasons we might try to fill it with likewise the same content as in the XML.

How would it be to change the current:
user<usernumber>=<clienttag>,<username>

into the Diablo standard naming scheme like:
user<usernumber>=<clienttag>,[<charname>*]<username>[@realm[@game]]
(which wouldn't break any current non-realm/non-D2 solutions)

Games could be enriched from:
game<gamenumber>=<clienttag>,<gamename>

to:
game<gamenumber>=<clienttag>,<gamename>[@realm]
