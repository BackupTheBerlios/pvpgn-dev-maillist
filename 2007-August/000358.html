On 8/8/07, All4you <zhaowei_hn at yahoo.com> wrote:
>    Hi , i am using PVPGN 1.8.0 server with mysql. My server have about 50
> 000 users and 3000 user online at once. Now my server become slowly. i think
> i must change some value in bnetd.conf. I see some value like :
>
>  usersync = 300
>  userflush = 300
>  userstep = 50
>  hashtable_size =  99
>
>  Is this correct with my server ? Who can help me change those values for
> best performance ^^ , thanks a lot

For hashtable size always use prime numbers and they should be bigger
in your case, try a prime number arround your number of accounts. The
other values can be left alone.

In any case you have not said anything what what exactly is slower.
Just playing with some bnetd.conf tuning values doesn't make MySQL
faster if MySQL is the problem.

You should also try pvpgn 1.99.x code that has no more overhead from
usersync/userstep periodic operations.
