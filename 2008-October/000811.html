Summary: Issues with Stealthbot (BNLS clients)

Details: I created an account with Stealthbot and logged in perfectly every time even after rebooting server.  However, if I try to login with the same username and password in the actual game, I get the error for wrong password.  If I make an account with WC3, both WC3 and Stealthbot can log into it.  Something must be wrong with the password storage for accounts created via a BNLS client.  I had a close friend verify this with duplicated results.

Also, the bot seems to crash if it's in a clan.  If I accept an invite on the bot, it crashes.  If I log in with an account that's in a clan, it crashes.  If that account is booted from the clan by cheiftain, it can log in again and work.  This behaviour does not occur on the official server leading me to believe PvPGN may be sending bunk data to Stealthbot that it doesn't know how to handle.

If you wish to duplicate this issue exactly, download and install the most popular battle.net bot here:
http://www.stealthbot.net/
And follow the instructions in this post to get the fixed EXE for War3 and fixed BNLS server entry:
http://www.stealthbot.net/board/index.php?showtopic=11542#Instructions

What loader have you been using for your WAR3 client? The "classic" loader or the SRP3-enabled loader?
