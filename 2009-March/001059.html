Summary: User Notification Message

Details: When there is a user notification message, such as one of your friends entering the server, joining a game, leaving a game, or leaving a server, the message is sent by the server name, instead of the user triggering the event, which Battle.net does.

For my server, Bored Aussie, this would be an example:
BoredAussie whispers: Your friend m1LkmaN has entered a game called "bvd".

Where it should be:
m1Lkman whispers: Your friend m1LkmaN has entered a game called "bvd".

This allows the /r command to go to the user triggering the event.

This can be fixed by changing line 184 of watch.cpp from:

message_send_text(dest_c,message_type_whisper,NULL,msg);

to

message_send_text(dest_c,message_type_whisper,my_c,msg);

I have tested this and it works great.


Thanks,
Jacob Godbout
