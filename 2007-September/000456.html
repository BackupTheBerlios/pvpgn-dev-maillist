You should not get the error in all those cases (assuming that the
proper library is linked in). Basically a library on windows has to be
made in some way for vs.net and another for mingw otherwise you get
such errors. The ones offered from gnuwin32 zlib for example do not
work with ming (but they do work with vs.net as someone is using them
as such on this list).

Someone on this list which builds pvpgn on windows often should help
you more (aaron?), I am more of a Unix guy.
