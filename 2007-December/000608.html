On Tuesday 04 December 2007 10:55:38 Ender wrote:
> I hate to put everything on my own shoulders but i will say i honestly
> have some plans to use pvpgn for my own game, (like planned on hopefully
> soon, within 6 months) and i would like nothing more than to built a
> video game on top of pvpgn, and keep my game server code closed, but
> still give back changes to pvpgn and bug fixes. This happens ALL the
> time btw in the BSD community. Companys base there products on Freebsd
> for example and when they find bugs and stuff they give back patches,
> "just because".

In this specific case you could provide full sources to your own pvpgn 
modifications (thus to the server part to support your game) and implement 
from 0 your own client side network code (which you had to do anyway because 
pvpgn does not have almost any client code). Why would you need to keep 
closed source the server part?

PvPGN is already "GPLv2 or later", there is no "turning back" to BSD license 
possible. Also, GPL licenses (those which force you to give back code) are 
not very attractive as you said for businesses to use (ie to use existent 
pvpgn code) but they are atractive to businesses when they want to release 
their own proprietary code to open source (since then they at least that any 
other competitive business has to provide sources to it if it will use their 
released code). So even with businesses and using GPL it depends...
