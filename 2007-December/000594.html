Summary: PostgreSQL tables and columns in lowercase

Details: The tables and columns in a PostgreSQL-Database are created lowercase, even if they're written in uppercase in sql_DB_layout.conf. When saving data into the tables all columns are created with uppercase-letters again, so every column that contains uppercase-letters (Like SEXP_0_losses and so on) is created twice and only the one with uppercase-letters is used.
I guess it's cause the SQL-commands to create the tables on first startup are without " around the column-names, you have to use "SEXP_0_losses" instead of SEXP_0_losses for example.

For detailed info, follow this link:
http://developer.berlios.de/bugs/?func=detailbug&bug_id=12591&group_id=2291
