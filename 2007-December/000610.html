On Tuesday 04 December 2007 16:38:25 Ender wrote:
> dizzy wrote:
> > On Tuesday 04 December 2007 10:55:38 Ender wrote:
> >> I hate to put everything on my own shoulders but i will say i honestly
> >> have some plans to use pvpgn for my own game, (like planned on hopefully
> >> soon, within 6 months) and i would like nothing more than to built a
> >> video game on top of pvpgn, and keep my game server code closed, but
> >> still give back changes to pvpgn and bug fixes. This happens ALL the
> >> time btw in the BSD community. Companys base there products on Freebsd
> >> for example and when they find bugs and stuff they give back patches,
> >> "just because".
> >
> > In this specific case you could provide full sources to your own pvpgn
> > modifications (thus to the server part to support your game) and
> > implement from 0 your own client side network code (which you had to do
> > anyway because pvpgn does not have almost any client code). Why would you
> > need to keep closed source the server part?
>
> Because these days a lot of "game content" is in the server side code.
> 100% dynamic events, real world changes like weather, and things like
> that. If you know Final Fantasy 11 (there first final fantasy MMO) there
> is a massive amount of "ecosystem" code that is all server side.
> Countrys trade with each other and effect prices, and weather effects
> your stats and all kinds of weird things like that. Server side content
> can be big and important.

Ok, then you could take another aproach. Have your content stuff in a separate 
server that talks with pvpgn a network protocol (just like d2gs does with 
d2cs and like you can see d2cs is GPL while d2gs is not nor does it needs to 
be). I think that won't change with AGPL since for all users communicating 
with pvpgn you can offer them your (modified) pvpgn source but you do not 
need to offer them your "game server" source because that one is not covered 
by AGPL nor does it needs to be (AGPL just as GPL does not "infect" programs 
that communicate with AGPL/GPL programs through network protocols and such).
