Well. Your "bug report" is not bug at this time but is feature request. The problem is as you sayed in address translations of player IPs. This feature IS NOT supported at this time for player IP addresses but ONLY for server IP address. I will try to code it in future.

For all: at this time is possible to play on server beside the NAT but 100% will work only two situations: 1)all of players have private IP and can connect to server without any NAT,
2) all of players have public IP (server can work in beside NAT but need to have right configured address_translation.conf)
