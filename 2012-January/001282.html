2011/12/15 Erik Hurtado Sepúlveda <erik.hurtado at mecsoft.cl>:
> As far as I know, this problem happens when a PVPGN server receives a
> malformed TCP packet, this happen to my server when we just start to host
> it(from another server), we determine the ip address that send the packet
> but i can't have the possibility to identify or capture a live dump when
> they happen.

You probably mean a malformed bnet "record" (not really TCP/IP packet,
since all that is handled in the OS). This is likely but we need to
determine which packet does that. A way to get pvpgn to dump all the
traffic (would make the server slower but at least you can see the
last packets received before it crashed) is to start it with the "-d"
option.

Another way would be to force it to actually crash instead of handling
out of memory as it does to get a core dump and then be able to run
gdb on it and see where exactly it crashed. I don't remember now what
changes in the code or compilation flags are needed to disable its OOM
handling.
