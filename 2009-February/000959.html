Submitted by: u-238
Assigned to : pandaemonium
Summary: Security exploit - disconnect users trying to view cg list

Details: This security exploit involves using a hosting bot to create a custom game where the game map path is longer than 128 characters.

This causes all Warcraft III clients to crash when they attempt to view the custom game list.

We have successfully fixed this exploit on Bored Aussie, as described below:


In "common/field_sizes.h":

Change:

const int MAX_GAMEINFO_LEN = 256;

To:

const int MAX_GAMEINFO_LEN = 128;


This prevents anyone from using the exploit.


For older versions, I believe the line is in the same file but it is this:

#define GAME_INFO_LEN 256

change to:

#define GAME_INFO_LEN 128
