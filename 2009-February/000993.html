Summary: gameinfo doesn't work on started games

Details: This bug was introduced in r524, "add feature which allow start game with same name when another game is already started or done", by Pelish. /gameinfo went from gamelist_find_game to gamelist_find_game_available -- which won't work if the game has already started. Since r524 allows more than one game by the same name, I'm not sure of the easiest way to fix this. Maybe just revert back to gamelist_find_game and then /gameinfo will then show for the first game by that name.

My own extensions to PVPGN allow game searching by gameid. Have a look at this example:
/where foo
Foo is using Warcraft III Frozen Throne and is currently in game "bar" (#001919)
/gameinfo 1919
..elided.. (same as /gameinfo bar)

gamelist_find_game_byid is already part of PVPGN. Maybe if you get a naming conflict when using /gameinfo, you can display all the matches and their IDs, and the user can then specify which one they want.
